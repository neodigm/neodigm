Rolling dice virtually is not the same experience as rolling dice physically. The dice do not appear to tumble or make sounds. This disconnected experience makes one suspect that maybe these on-line dice are not really random.

It would be great to create the same experience as actually rolling dice. It would be like marathon Monopoly with your friends. It would be like Vegas baby!

In this article I introduce a gamified micro-interaction in which dice are virtually rolled by shaking the phone. The sensation is complete with haptic feedback making it seem like the bones are rattling in your sweaty palm. The mathematical integrity of the roll results is achieved through the high entropy of the Web Crypto API.

"Why?", you might and should ask. I am doing this because it allows me to integrate emerging technologies in a way that I have a slick deliverable that I can demonstrate when I am done. In the Midwest we call this "cool beans".

You will learn these skills while building this project:

  • Accelerometer API
  • Audio API
  • Blender 3D Modeling
  • Git
  • glTF
  • HTML canvas
  • JavaScript
  • NPM
  • Three.js
  • Vibration API
  • UV Mapping
  • Web Crypto API
  • WebGL
  • Webpack

This is totally doable for you because you are a smart developer, designer, and magical unicorn.
